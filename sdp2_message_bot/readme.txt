sdp2_message_bot — Linux

Назначение
----------
1) Основное: фоновая проверка выхода в интернет каждые 2 минуты (настраивается).
   Успешные проверки в лог не пишутся; раз в 5 часов — одна строка «всё хорошо».
   Если основная проверка не прошла — ещё 3 проверки с интервалом 20 с; если все
   провалы — в лог пишется короткая сетевая диагностика, затем сброс сетевого
   интерфейса (ip link или nmcli). Если связи нет —
   создаётся флаг и выполняется reboot. После сброса интерфейса или после ребута в
   целевой чат Telegram уходит короткое уведомление.

2) Дополнительно: пересылка всех сообщений из групп и каналов в TARGET_CHAT_ID
   (личные чаты не обрабатываются).

3) Команда «Матрица перезагрузка»: если в тексте или подписи сообщения есть фраза
   триггера (по умолчанию
   «матрица перезагрузка», без учёта регистра), в целевой чат отправляется текст
   «Кассета жрет магнитную ленту», затем перезагрузка сервера. После загрузки в чат
   уходит отдельное сообщение (TELEGRAM_MSG_AFTER_MANUAL_REBOOT). Для альбома
   достаточно триггера в подписи любого кадра.

Права
-----
Сброс интерфейса и перезагрузка по умолчанию идут через sudo без пароля (-n):

  sudo -n /usr/bin/ip link set <NETWORK_IFACE> down
  sudo -n /usr/bin/ip link set <NETWORK_IFACE> up
  sudo -n /usr/sbin/reboot

В /etc/sudoers.d/ для пользователя службы (например sdp-server) нужны NOPASSWD-строки
с теми же абсолютными путями и именем интерфейса, что в .env.

Пути к бинарникам при необходимости задаются в .env: IP_BIN, REBOOT_BIN, NMCLI_BIN.
Если бот запускается от root и sudo не нужен: PRIVILEGED_USE_SUDO=0.

При NIC_RESET_METHOD=nmcli команды nmcli тоже вызываются через sudo -n — добавьте их в sudoers.

Переменные .env (обязательные для Telegram)
--------------------------------------------
TELEGRAM_BOT_TOKEN
TARGET_CHAT_ID

MATRIX_REBOOT_TRIGGER           — подстрока-команда ребута, по умолчанию матрица перезагрузка
MATRIX_REBOOT_SUBSTITUTE_TEXT   — что отправить в чат вместо пересылки, по умолчанию «Кассета…»
TELEGRAM_MSG_AFTER_MANUAL_REBOOT — текст после ребута по команде Матрица
POST_MANUAL_REBOOT_NOTIFY_FLAG  — путь к флагу ручного ребута (по умолчанию рядом со скриптом)

Переменные .env (watchdog)
--------------------------
NETWORK_IFACE           — имя интерфейса, например eth0 или enp0s3 (желательно задать)

PRIVILEGED_USE_SUDO     — 1/0, по умолчанию 1 (sudo -n для ip link и reboot)
IP_BIN                  — по умолчанию /usr/bin/ip
REBOOT_BIN              — по умолчанию /usr/sbin/reboot
NMCLI_BIN               — по умолчанию /usr/bin/nmcli

WATCHDOG_ENABLED        — 1/0, по умолчанию 1
INTERNET_CHECK_URL      — URL проверки, по умолчанию https://1.1.1.1/cdn-cgi/trace
INTERNET_CHECK_URL_ALT  — опционально запасной URL
INTERNET_CHECK_INTERVAL_SEC — по умолчанию 120
INTERNET_CHECK_TIMEOUT_SEC  — таймаут HTTP, по умолчанию 5
WATCHDOG_FAST_RETRY_SEC     — пауза между быстрыми повторами при сбое, по умолчанию 20
WATCHDOG_FAST_RETRY_COUNT   — число быстрых повторов после первой неудачи, по умолчанию 3
WATCHDOG_HEARTBEAT_SEC      — раз в столько секунд писать в лог что всё ОК, по умолчанию 18000 (5 ч)
NIC_RESET_METHOD        — ip (по умолчанию) или nmcli
POST_NIC_WAIT_SEC       — пауза после сброса интерфейса перед новой проверкой, по умолчанию 45
POST_REBOOT_NOTIFY_FLAG — путь к файлу-флагу уведомления после ребута (по умолчанию рядом со скриптом)
DIAG_LOG_MAX_CHARS      — лимит символов диагностики в логе, по умолчанию 5000
STARTUP_NOTIFY_RETRIES  — попыток отправить сообщение после ребута, по умолчанию 3
STARTUP_NOTIFY_RETRY_DELAY_SEC — пауза между попытками, по умолчанию 30
TELEGRAM_MSG_AFTER_NIC_RESET   — текст после успешного сброса интерфейса
TELEGRAM_MSG_AFTER_REBOOT      — текст после загрузки, если был ребут по watchdog

Логи
----
bot.log, ротация в полночь, хранение 30 файлов.
